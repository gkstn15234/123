구글 디스커버 타겟 뉴스 사이트 "이콘밍글" PRD (Product Requirements Document)
1. 개요 (Overview)
본 문서는 구글 디스커버 및 구글 뉴스 노출을 최우선 목표로 하는 경제 뉴스 사이트 **'이콘밍글(Econmingle)'**의 MVP(Minimum Viable Product, 최소 기능 제품) 개발을 위한 제품 요구사항 명세서입니다. '이콘밍글'은 독자들에게 시의성 있고 깊이 있는 경제 뉴스를 빠르고 신뢰도 높게 전달하는 것을 목표로 합니다.

2. 기술 스택 및 배포 워크플로우
콘텐츠 관리 및 생성 (Local): Hugo

마크다운(.md) 형식으로 기사를 작성하여 빠르고 쉽게 콘텐츠 생성.

로컬 서버에서 실시간으로 기사 최종본 미리보기.

버전 관리 및 소스코드 저장소 (Remote): GitHub

모든 콘텐츠와 사이트 코드를 저장하고 버전을 관리.

배포 및 호스팅 (Live): Cloudflare Pages

GitHub 저장소의 특정 브랜치(예: main 또는 master)에 변경사항이 Push되면, 자동으로 빌드 및 배포(CI/CD)를 수행.

Cloudflare의 전 세계 CDN을 통해 사용자에게 매우 빠른 속도로 페이지 제공.

워크플로우 상세:

기사 작성 (기자):

로컬 PC에 설치된 Hugo 환경에서 VSCode, Obsidian 등의 에디터를 사용해 마크다운으로 기사 작성.

hugo server 명령어로 로컬에서 기사 레이아웃 및 내용을 최종 확인.

업로드 (기자/에디터):

작성이 완료된 기사 파일을 git 명령어나 GitHub Desktop 툴을 사용하여 GitHub 저장소에 Push.

(추가 기능 제안) "깃허브에 올리기" 버튼이 포함된 간단한 내부 스크립트 또는 애플리케이션을 제작하여 클릭 한 번으로 git add, commit, push 과정을 자동화.

자동 빌드 및 배포 (시스템):

Cloudflare Pages는 GitHub 저장소의 변경을 감지.

즉시 Hugo 빌드를 시작하여 정적 HTML 파일들 생성.

빌드가 완료되면 새로운 버전을 전 세계 Cloudflare CDN 네트워크에 배포. 배포 시간은 수십 초에서 몇 분 내외.

가. 회사소개
이콘밍글(Econmingle)을 찾아주셔서 감사합니다.

이콘밍글은 'Economy'와 'Mingle'의 합성어로, 복잡하고 어려운 경제 소식을 독자들의 삶과 자연스럽게 어우러지도록 만들겠다는 비전을 담고 있습니다.

저희는 IT 기술의 발전, 급변하는 국제 정세, 국내 정책 변화 등 경제에 영향을 미치는 다양한 요소를 심도 있게 분석하고, 독자들이 합리적인 경제적 의사결정을 내리는 데 도움이 되는 인사이트를 제공하고자 합니다.

정확하고 빠른 뉴스: 사실에 기반한 정확한 정보를 가장 빠르게 전달합니다.

깊이 있는 분석: 현상의 이면을 꿰뚫는 깊이 있는 분석과 전망을 제시합니다.

독자 중심의 콘텐츠: 독자의 눈높이에서 가장 궁금해하는 주제를 찾아 알기 쉽게 풀어냅니다.

이콘밍글은 주식회사 픽펄스가 만드는 새로운 시대의 경제 미디어입니다. 독자 여러분의 목소리에 항상 귀 기울이며 함께 성장해 나가겠습니다.

마. 편집 지침
이콘밍글은 언론으로서의 사회적 책임을 다하고 독자의 신뢰를 얻기 위해 다음과 같은 편집 지침을 준수합니다.

사실과 진실 보도: 모든 기사는 사실에 근거하여 작성하며, 진실을 공정하게 보도할 의무를 가집니다.

독립성: 외부의 어떠한 압력이나 청탁에도 흔들리지 않고, 오직 독자와 사회의 이익을 위해 독립적으로 편집권을 행사합니다.

인권 존중 및 사생활 보호: 취재 및 보도 과정에서 개인의 인권을 존중하고, 공공의 이익과 무관한 사생활을 침해하지 않습니다.

오보의 정정: 보도 내용에 오류가 발견될 경우, 이를 신속하게 인정하고 독자에게 명확하게 알린 후 정정합니다.

5. SEO 및 사이트맵 전략
구글 검색엔진이 사이트의 구조와 콘텐츠를 효과적으로 이해하고 수집할 수 있도록 다음의 사이트맵을 생성하고 robots.txt에 명시합니다.

일반 사이트맵 (/sitemap.xml):

모든 페이지(홈, 카테고리, 기자, 기사, 정적 페이지)의 목록을 포함하는 표준 사이트맵.

구글 뉴스 사이트맵 (/news-sitemap.xml):

최근 48시간 이내에 발행된 기사만 포함.

<publication>, <publication_date>, <title> 등 구글 뉴스 전용 태그를 포함하여 기사 발행 정보를 명확히 전달.

로컬 사이트맵 (Geo Sitemap) (/geo-sitemap.xml):

언론사의 물리적 위치 정보를 제공하여 지역 기반 검색에 대응.

KML 또는 GeoRSS 형식을 사용하여 회사 주소와 좌표(위도, 경도)를 명시.

기자 사이트맵 (/author-sitemap.xml):

모든 '기자별 페이지'의 URL을 포함. 구글이 각 기자의 전문성(E-E-A-T)을 인지하고 기사의 신뢰도를 높이는 데 도움을 줌.

동영상 사이트맵 (/video-sitemap.xml):

(향후 확장 기능) 기사에 포함된 동영상 콘텐츠의 썸네일, 제목, 설명, 재생 시간 등의 정보를 제공하여 동영상 검색 결과 노출을 돕는다. MVP에서는 빈 파일 또는 생성하지 않음.

페이지 사이트맵 (/page-sitemap.xml):

이용약관, 개인정보처리방침 등 정적 페이지의 목록을 별도로 관리. 콘텐츠 업데이트 빈도가 낮은 페이지들을 분리하여 크롤링 효율을 높임.
